<objective>
Remove `#![allow(missing_docs)]` from all 42 Rust source files and add `///` doc comments to all public items, prioritizing Tauri commands (IPC boundary), core public types/traits, and key data structures.

Purpose: Enforce documentation-as-usual in the codebase. Tauri commands form the IPC boundary between Rust backend and React frontend; they must be documented so that maintainers understand the contract without reading implementation. Core types (AppError, models) are consumed across the entire codebase and need docs for discoverability.
Output: 42 Rust files with `#![allow(missing_docs)]` removed and appropriate doc comments added.
</objective>

<context>
Block 01 (dead code removal) must complete first since CLEAN-01 modifies some of the same files. This plan operates on a codebase where dead code has been eliminated.

**Strategy by tier:**

- **Tier 1 — IPC Boundary (MUST have doc comments, no exceptions):** All `#[tauri::command]` annotated functions across `commands/` directory. Each command is called by the React frontend via `invoke()`. The doc comment must describe: what the command does, the arguments it expects, and what it returns (and what success/error looks like).

- **Tier 2 — Core Public API (MUST have doc comments, no exceptions):** All `pub struct`, `pub enum`, `pub trait`, and `pub fn` in `core/` and `models/` that are re-exported or used across module boundaries. Key items: `AppError`, `AppResult`, `Database`, `AITaskQueue`, `InferenceProvider`, `Image` model, `Task` model, `ImageCategory`, `VectorEntry`, `SearchResult`, `BruteForceVectorIndex`, `FaceDetector`, `ClipEmbedder`, `OnnxRuntimeManager`, `KnowledgeGraphEngine`, `CircuitBreaker`.

- **Tier 3 — Internal/utils (doc comments OR selective allow):** Utilities (`hash.rs`, `crypto.rs`, `error.rs`, `log_sanitizer.rs`). These are consumed internally. If a module has only internal items that are well-named, add minimal one-line docs to pub items. If a module exports no pub items, replace the file-level `#![allow(missing_docs)]` with a module-level doc comment (`//! Module docs`) and no allow attribute needed.

**DON'Ts:**
- Do not write overly long docs. One or two sentences suffice. Example good doc: `/// Validates and sanitizes file paths to prevent directory traversal attacks.`
- Do not add `#[allow(missing_docs)]` back anywhere at the file level (except internal-only modules with no pub items, where `//!` module doc is preferred).
- Do not add doc comments to private items or test-only items.
- Do not change any behavior, add any code, or remove any code.

**DOs:**
- For each `#[tauri::command]`, examine the function signature to understand the parameters and return type, then write a doc that describes the IPC contract.
- For `pub struct`/`pub enum`, describe the role of the type.
- For `pub fn` in a `pub impl` block, describe the method's purpose.
- Where a module truly has no public items and is internal-only, replace the allow attribute with `//! Module docs` and add `#![doc(hidden)]` if it should be excluded from generated docs.
</context>
