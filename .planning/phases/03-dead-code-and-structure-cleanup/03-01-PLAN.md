<context>
This plan removes ~54 instances of `#[allow(dead_code)]` across 20 files, plus two fully orphaned functions (`init_database`, `try_open_database`, `is_encrypted`).

**Dead code categories:**
1. **Enum variants** — Unused variants in `VectorIndexError`, `FaceError`, `QueueCommand` (remove variant + Display arm)
2. **Struct fields** — Unused fields like `index_dir`, `command_sender`, `clip_embedder`, `vector_index`, `edge_thresholds` (remove field + constructor)
3. **Methods** — Unused public/private methods (`batch_insert`, `get`, `contains`, `count`, `get_dimension`, `clear`, `save_to_file`, `load_from_file`, `batch_classify`, `get_imagenet_classes`, `batch_embed_images`, `compute_similarity`, `find_similar_faces`, `unregister_face`, `input_shape`, `output_dim`, `get_models_dir`, `list_available_models`, `set_concurrency`, `cancel`, `failure_count`, `has_conflicts`, `delete_for_image`, `hamming_distance`, `model()`, `not_found()`, `auth()`, `redact_api_key`, `redact_path`, `redact_url`, `init()`)
4. **Whole structs** — `SanitizedDisplay`, `SimilarityResult` (remove struct + impl)
5. **Test helper functions** — `setup_test_db` in search.rs and batch_ops.rs: replace `#[allow(dead_code)]` with `#[cfg(test)]` gate (actually they're already in `#[cfg(test)]` mod, so just remove the allow attribute)
6. **Struct-level allows** — `#[allow(dead_code)]` on `Image` and `Task` structs: the structs are used but `from_row` methods are unused or the structs exist only for deserialization. These should get `#[expect(dead_code)]` or be individually evaluated.

**Architectural note:** The GSD workflow is not using Cargo features (`cfg(feature = "...")`). None of the dead code items are feature-gated stubs. All are simply unused. Clean deletion is the correct disposition.

**Verification strategy:** After each file modification, check compilation. Final verification runs full cargo check + clippy + existing tests.
</context>
