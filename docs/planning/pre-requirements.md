## 📋 项目概览

### 项目名称
**ArcaneGallery** - 本地AI驱动的图片知识库管理系统

### 电梯演讲
一个面向个人用户的本地图片知识库，通过AI图像识别技术实现智能分类与检索，支持万张级别图片管理，完全本地化部署确保隐私安全，提供专业级的元数据管理和多维度搜索能力。

### 核心价值主张
- **隐私优先**: 所有数据本地存储，AI处理本地运行
- **智能管理**: AI自动识别、标签、分类
- **专业级**: 支持RAW格式、完整元数据管理
- **高性能**: 万张图片秒级检索
- **易用性**: 现代化界面，流畅交互体验

## 🎯 目标用户画像

### 主要用户群体
**个人专业用户** - 摄影爱好者、设计师、内容创作者

#### 用户特征
- 年龄: 25-45岁
- 技术能力: 中高级
- 设备: Windows/macOS/Linux桌面设备
- 图片规模: 5,000-50,000张
- 痛点: 图片太多找不到、管理混乱、缺乏组织

#### 典型场景
1. **摄影师**: 拍摄大量RAW照片，需要按拍摄时间、地点、主题快速检索
2. **设计师**: 管理设计素材库，需要按颜色、风格、类型分类
3. **内容创作者**: 管理社交媒体素材，需要快速找到特定主题图片
4. **旅行爱好者**: 按地理位置、时间线整理旅行照片

## 🏆 对标产品分析

### 顶尖产品对比

| 产品 | 核心优势 | 适用场景 | 技术特点 |
|------|---------|---------|---------|
| **PhotoPrism** | 开源本地AI，隐私优先 | 技术用户，自建服务器 | Docker部署，MariaDB，PWA界面 |
| **Adobe Lightroom** | 专业编辑+管理 | 专业摄影师 | 非破坏性编辑，RAW支持，Adobe Sensei AI |
| **Google Photos** | 云端AI，易用性强 | 普通用户 | 自然语言搜索，人脸识别，15GB免费 |

| **LibrePhotos** | 开源私有云 | 企业/团队 | PostgreSQL，人脸聚类，语义搜索 |

### 最佳实践总结

#### 1. **元数据管理** (参考: Adobe Lightroom, ExifTool)
- 支持三大标准: EXIF (技术参数) + IPTC (版权信息) + XMP (可扩展元数据)
- 自动提取相机参数: 光圈、快门、ISO、焦距、拍摄时间
- GPS定位信息提取与地图展示
- 批量编辑元数据 (版权、作者、关键词)
- XMP sidecar文件支持 (RAW文件非破坏性编辑)

#### 2. **AI能力** (参考: Google Photos, PhotoPrism, Apple Photos)
- **图像内容识别**: 自动识别物体、场景、人物
  - 物体识别: 动物、车辆、食物、建筑、自然景观
  - 场景识别: 海滩、山脉、城市、室内、夜景
  - 人物识别: 人脸检测与聚类 (可选功能)
- **自动标签生成**: 基于图像内容自动生成描述性标签
- **智能分类建议**: 根据图片内容推荐分类
- **语义搜索**: 支持自然语言查询 (如"夕阳下的海滩")

#### 3. **搜索功能** (参考: Google Photos, Adobe Lightroom)
- **多维度搜索**:
  - 文件名搜索
  - 关键词标签搜索
  - 元数据筛选 (相机型号、镜头、拍摄时间范围)
  - GPS地理位置搜索
  - AI内容搜索 (识别到的物体、场景)
- **高级筛选**:
  - 组合条件搜索 (AND/OR逻辑)
  - 评分筛选 (1-5星)
  - 颜色标签筛选
  - 文件类型筛选
- **视觉搜索**:
  - 相似图片搜索 (以图搜图)
  - 颜色相似度搜索

#### 4. **组织与分类** (参考: Adobe Lightroom, Apple Photos)
- **多级文件夹结构**: 按年/月/日或自定义层级
- **智能相册**: 基于规则自动更新 (如"所有5星照片")
- **手动相册**: 用户自定义相册集合
- **标签体系**:
  - 层级标签 (如: 旅行 > 日本 > 东京)
  - 标签云可视化
  - 标签使用频率统计
- **评级系统**: 1-5星评级 + 颜色标签
- **收藏夹**: 快速访问常用图片

#### 5. **格式支持** (参考: PhotoPrism, Adobe Lightroom)
- **常见格式**: JPEG, PNG, WebP, GIF, BMP, TIFF
- **RAW格式**: CR2 (Canon), NEF (Nikon), ARW (Sony), DNG, RAF (Fuji), ORF (Olympus)
- **现代格式**: HEIC/HEIF (Apple), AVIF
- **视频支持** (可选): MP4, MOV, AVI

#### 6. **预览与管理界面** (参考: Google Photos, Apple Photos)
- **缩略图系统**:
  - 多尺寸缩略图生成 (64px, 256px, 1024px)
  - 渐进式加载
  - 缩略图缓存策略
- **视图模式**:
  - 网格视图 (可调节缩略图大小)
  - 列表视图 (显示详细元数据)
  - 全屏预览
  - 地图视图 (GPS定位)
- **批量操作**:
  - 批量标签、评级、移动、删除
  - 批量导出 (支持调整尺寸、格式转换)
  - 批量重命名
- **文件信息面板**:
  - EXIF信息展示
  - IPTC/XMP元数据编辑
  - 文件大小、尺寸、格式信息

#### 7. **数据备份与恢复** (参考: PhotoPrism)
- **备份策略**:
  - 完整备份 (数据库 + 图片文件)
  - 增量备份 (仅变更部分)
  - 加密备份 (AES-256-GCM)
- **版本历史**: 保留元数据修改历史
- **导入导出**:
  - 支持从其他图库导入 (Lightroom, Apple Photos)
  - 导出为标准格式 (保留元数据)

#### 8. **权限控制** (针对未来多用户扩展)
- **用户角色**: 管理员、编辑者、查看者
- **相册权限**: 公开、私有、指定用户可见
- **操作日志**: 记录用户操作历史

#### 9. **扩展接口**
- **API接口**: RESTful API供第三方应用调用
- **插件系统**: 支持自定义扩展
- **命令行工具**: 批量操作脚本支持

## 🎨 核心功能模块

### 模块1: 图片存储系统
**优先级**: P0 (核心必需)

#### 功能要点
- 支持用户指定存储目录
- 自动检测并导入新增图片
- 文件命名规范: `{日期}_{原始文件名}` 或自定义模板
- 重复文件检测 (基于文件hash)
- 文件完整性校验

#### 技术规范
- 存储路径: 用户可配置，默认 `{用户文档}/ArcaneGallery/`
- 文件组织: 保持原始目录结构或按日期重组
- 支持外部存储: NAS、网络驱动器挂载

---

### 模块2: 元数据管理
**优先级**: P0 (核心必需)

#### 功能要点
- 自动提取EXIF、IPTC、XMP元数据
- 批量编辑元数据 (版权、作者、关键词)
- GPS定位信息提取与地图展示
- XMP sidecar文件支持 (RAW文件)
- 元数据搜索与筛选

#### 技术规范
- 使用 `kamadak-exif` (Rust) 或 `exiftool` 提取EXIF
- 使用 `rexiv2` (Rust) 处理IPTC/XMP
- 地图服务: 集成OpenStreetMap或离线地图

---

### 模块3: AI图像识别
**优先级**: P1 (重要功能)

#### 功能要点
- 本地AI模型运行 (无需联网)
- 自动识别物体、场景、人物
- 自动生成描述性标签
- 人脸检测与聚类 (可选)
- 智能分类建议

#### 技术规范
- 使用ONNX Runtime或TensorFlow Lite
- 模型选择: MobileNet、EfficientNet、CLIP
- 人脸识别: InsightFace或FaceNet
- 模型存储: 本地模型文件，首次使用时下载

### 模块4: 搜索引擎
**优先级**: P0 (核心必需)

## 🛠️ 技术栈选择

### 后端 (Rust + Tauri)
- **框架**: Tauri 2.x
- **数据库**: SQLite (sqlx) + FTS5全文搜索
- **图像处理**: image, kamadak-exif, rexiv2
- **AI推理**: onnxruntime或tch-rs (PyTorch Rust绑定)
- **搜索引擎**: Tantivy或SQLite FTS5
- **地图**: 离线地图或OpenStreetMap API

### 前端 (React + TypeScript)
- **框架**: React 18 + TypeScript
- **状态管理**: Zustand (沿用现有)
- **UI组件**: Tailwind CSS + shadcn/ui (沿用现有)
- **虚拟滚动**: react-window
- **地图**: react-leaflet
- **图标**: Lucide React

### AI模型
- **图像分类**: MobileNetV3或EfficientNet-Lite
- **目标检测**: YOLOv8-nano (轻量级)
- **人脸识别**: InsightFace (可选)
- **语义搜索**: CLIP (可选，用于自然语言搜索)

## 📊 性能要求

### 响应时间
- 缩略图加载: < 100ms (缓存命中)
- 搜索响应: < 500ms (万张图片)
- 图片导入: < 2s/张 (含元数据提取)
- AI识别: < 5s/张 (本地CPU)

### 存储优化
- 缩略图缓存: 多级缓存 (内存 + 磁盘)
- 数据库索引: 时间、标签、GPS、文件名
- 压缩: WebP格式缩略图 (质量85%)

### 并发处理
- 后台索引服务: 不影响前台操作
- 批量操作: 支持取消和进度显示
- AI处理队列: 异步处理，优先级队列

---

## 🔐 安全与隐私

### 数据安全
- 本地存储: 所有数据存储在用户本地设备
- 加密备份: AES-256-GCM加密
- 敏感信息: GPS定位可选择性清除
- 无联网要求: 核心功能完全离线可用

### 隐私保护
- AI本地运行: 不上传图片到云端
- 元数据保护: 支持清除敏感EXIF信息
- 用户控制: 完全掌控自己的数据
